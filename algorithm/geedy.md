# Geedy

- 지금 당장 좋아 보이는 선택을 하는 알고리즘
- 매 순간 최선의 선택을 하면 전체적으로도 최적이 될거라고 믿는 알고리즘

## 핵심개념
- 매 선택에서 '즉각적으로 가장 좋은 선택'을 한다.
  - 전체 최적을 고려하는게 아니라, 그 순간 가장 이득이 되는 선택을 반복
  - 뒤를 돌아보지 않고 앞으로만 감

| 조건                         | 설명                               |
| -------------------------- | -------------------------------- |
| **Greedy Choice Property** | 지금 선택한 최선이 전체 해에서도 포함되는가?        |
| **Optimal Substructure**   | 문제를 쪼개도 부분 문제의 해가 전체 최적해에 포함되는가? |


### Greedy Choice Property (그리디 선택 조건)
- “지금 당장 최선의 선택이, 결국 전체 해에서도 올바른 선택이어야 한다." -> 되돌아보지 않는 알고리즘


### Optimal Substructure (최적 부분 구조)
- “문제를 잘라놓아도, 부분 문제의 최적해가 전체 최적해에 포함된다.”

```markdown

✅ 문제: 최소 화폐 교환 문제 (단순 버전 아님)

어떤 나라에서는 아래와 같은 비정규 화폐 단위를 사용한다.

화폐 단위: 1, 4, 6, 9


이 나라에서 금액 n원을 거슬러 주려고 한다.
단, 동전 개수를 최소로 사용해야 한다.

예를 들어

n = 8 → 4 + 4 = 2개

n = 7 → 6 + 1 = 2개

```
```python
def solution(n):
    coin = [9, 6, 4, 1]
    cnt = 0
    for c in coin:
        while n >= c:
            cnt += 1
            n = n - c
    return cnt

solution(18) # 그리디의 이점
```

## 장점
- 시간복잡도가 O(N log n) + 단순 반복(O(N)) 수준으로 빠름
- 구현이 쉽고 단순함
- 메모리 사용이 거의 없다

## 단점
- 한 번 내린 선택을 되돌릴 수 없다.
- 가장 빨리 좋아보이는 선택만 하고 선택을 되돌릴 수 없기에 앞으로 선택이 가능한 상황을 놓칠수 있음
  > solution(12): 6 + 6 으로 2개의 동전이 가장 최적의 선택이지만, greedy 알고리즘의 한계때문에, 9 + 1 + 1 + 1 로 4개가 나옴
